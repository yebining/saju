# "한 입" 컨셉 강화 — 오늘의 한 입 + 한 입 더 (초안 설계)

> 2026-06-05 · 사주 허브에 "한 입(snackable)" 컨셉을 입혀 **재방문 습관**과 **프리미엄 진입 구조**를 잡는 초안. 외부 키(AI/결제/DB) 없이 결정론적으로 구현.

## 1. 배경 & 방향

서비스 이름 "사주 한 입"의 컨셉을 제품에 반영하기로 결정(A안: 한 입 강화). 핵심 두 기능을 **기존 UI에 추가만** 한다 — 기존 화면·흐름·디자인은 변경하지 않는다.

- **오늘의 한 입** — 매일 바뀌는 미니 운세 → 재방문/습관 루프.
- **한 입 더** — 기본 결과 아래에 더 깊은 풀이를 무료로 펼침 → 프리미엄으로 가는 다리.

## 2. 수익 모델 (구조만 — 실제 결제는 이후)

**넓이는 다 열고, 깊이를 판다 (프리미엄).** 카테고리는 전부 무료로 맛보고, 결제는 가장 깊은 층에만 붙인다. "한 입 → 한 입 더 → 한 상"의 음식 사다리:

| 단계 | 내용 | 과금 |
|---|---|---|
| 🍬 오늘의 한 입 | 매일 운세 | 무료 |
| 🥄 한 입 (기본 결과) | 점수·오행·강점/주의점 (현행) | 무료 |
| 🥢 한 입 더 | 조금 더 깊은 풀이 | **무료** (귀여운 떡밥 유지) |
| 🍱 사주 한 상 (정식 풀이) | 올해 흐름·시기별 운·구체 행동 처방·심층 궁합 | **유료 (이후)** |

이번 초안은 🍬🥄🥢까지 실제로 만들고, 🍱은 결과 맨 아래 **"준비 중" 티저 자리**만 잡는다.

## 3. 톤 규칙 (중요)

표면 텍스트는 전부 **다정한 "오늘의 운세" 말투** ("~하네요", "~하길 바랍니다"). "수생목" 같은 명리 용어는 **절대 노출하지 않는다.** 오행/일진 로직은 내부에서만 돌고, 사용자에겐 쉬운 문장으로만 보인다.

## 4. 기능 ① 오늘의 한 입 (홈 상단 추가)

### 동작
- 내 사주가 저장돼 있을 때만 홈 상단에 카드로 노출. **없으면 아무것도 안 띄움**(기존 MySajuCard의 "저장하기" 프롬프트가 그 역할 — 저장하면 자연스럽게 잠금 해제).
- 내용은 **오늘 날짜의 일진(日辰) × 내 일간**을 비교해 결정론적으로 생성. 매일 일진이 바뀌므로 매일 달라진다.

### 카드 구성
```
☀️ 오늘의 한 입
{오늘의 운세 한마디 — 서술형 2문장}

🎯 오늘의 작은 한 입
  → {오늘의 작은 결정/행동 넘지 한 줄}

🍙 오늘의 음식   {food}
🎨 오늘의 색깔   {color}
🫶 오늘의 사람   {person}
```

### 생성 로직 (`lib/saju/daily.ts`, 순수 함수)
- 입력: `{ myDayOhaeng: Ohaeng, todayDayOhaeng: Ohaeng, daySeed: number }`
  - `myDayOhaeng` = 내 일간(일주 천간)의 오행.
  - `todayDayOhaeng` = 오늘 일진(오늘 날짜의 일주 천간)의 오행.
  - `daySeed` = 오늘 일진의 천간·지지 인덱스로 만든 결정론적 정수(매일 다름; 음식/사람 변주에 사용).
- 출력 `DailyBite = { mood: string; nudge: string; food: string; color: string; person: string }`.
- **mood/nudge**: 두 오행의 관계(같음 / 내가 받는 생 / 내가 하는 생 / 내가 받는 극 / 내가 하는 극)를 5가지로 분류 → 각 분류별 다정한 문장 테이블에서 선택. (명리 용어 없음)
  - 예: "오늘은 주변에서 당신을 챙겨주는 기운이 흘러요. 무리하지 않아도 일이 술술 풀리는 하루가 되겠네요 🍀" / nudge: "미뤄둔 연락, 오늘 먼저 건네보기 좋은 날이에요."
- **color**: 오늘 일진 오행의 전통색 (목=청록, 화=다홍, 토=노랑, 금=흰색, 수=남색).
- **food**: 오늘 일진 오행별 음식 풀에서 `daySeed`로 선택 (목=비빔밥, 화=떡볶이… 식, 풀 3~4개).
- **person**: 공용 "오늘의 사람" 풀(오래된 친구 / 직장 선배 / 부모님 / 새로운 인연 / 가까운 후배 / 옛 동료)에서 `daySeed`로 선택.

### 컴포넌트 (`components/today-bite.tsx`, 클라이언트)
- `useEffect`에서: `loadMySaju()` → 없으면 `null` 렌더(아무것도 안 보임). 있으면:
  - 내 일주 = `calculateSaju(saved).day`.
  - 오늘 일주 = `calculateSaju({ year, month, day, hour:12, minute:0, isLunar:false, isLeapMonth:false }).day` (오늘 날짜는 `new Date()`로).
  - `daily.ts`로 DailyBite 계산 → state 저장.
- 초기 렌더는 `null`(SSR·하이드레이션 안전; `new Date()`/localStorage는 effect에서만 사용).
- 홈(`app/page.tsx`)에 `<TodayBite />`를 헤더 아래·MySajuCard 위에 **추가만** 한다.

## 5. 기능 ② 한 입 더 (결과 페이지 맨 아래 추가)

### 동작
- 기존 결과 섹션(점수·오행·강점/주의점·접이식 만세)은 **그대로**. 맨 끝에 새 블록만 추가.
- `🥢 한 입 더` — `<details>`로 접힌 상태. 펼치면 더 깊은 풀이가 **무료로** 노출.
- 그 아래 `🍱 사주 한 상 (정식 풀이)` — **비활성 티저 카드**: "곧 만나요 — 올해 흐름과 시기별 운까지 한 상 가득 차려드릴게요 (준비 중)". 실제 동작/결제 없음. 이후 프리미엄 잠금이 들어갈 자리.

### 생성 로직 (`lib/saju/reading-deep.ts`, 순수 함수)
- 입력: `(category: Category, count: OhaengCount)`.
- 출력 `DeepReading = { sections: { title: string; body: string }[] }` (2~3개 섹션).
  - 섹션 예: "요즘 당신의 흐름", "이런 결정이 잘 맞아요", "이 타이밍은 한 박자 쉬어가기".
  - 카테고리·오행 분포로 결정론적 분기, 전부 쉬운 말투. (지금은 더미 — 진짜 깊이는 Phase 2 AI에서)

### 컴포넌트 (`components/deeper-reading.tsx`)
- props: `{ category: Category; count: OhaengCount }`.
- `generateDeepBite`로 섹션 생성, `<details><summary>🥢 한 입 더</summary>…</details>`로 렌더 + 하단 🍱 티저 카드.
- 결과 페이지(`app/result/[id]/page.tsx`)에서 면책 문구 위에 `<DeeperReading category={view.category} count={view.meOhaeng} />` **추가만**.

기존 파일은 위 2곳에 컴포넌트 한 줄씩 추가하는 것 외에 **변경 없음**.

## 7. 데이터 흐름

- 오늘의 한 입: `localStorage(내 사주)` + `new Date()(오늘)` → `calculateSaju` ×2 → `dailyFortune()` → 카드. (서버/DB 무관, 전부 클라이언트)
- 한 입 더: 결과 페이지가 이미 가진 `meOhaeng`·`category` → `generateDeepBite()` → 펼침 섹션. (추가 fetch 없음)

## 8. 에러 처리

- 내 사주 없음 → `<TodayBite/>`는 `null` (조용히 미노출).
- `calculateSaju`가 throw할 일은 저장된 유효 입력에선 없음(저장 시 검증됨). 방어적으로 try/catch 후 미노출.
- 잘못된 오행 키 등은 타입으로 차단.

## 9. 테스트

- `daily.test.ts`: 고정된 (내 오행, 오늘 오행, daySeed)로 `dailyFortune` 호출 → mood/nudge/food/color/person이 비어있지 않고, 같은 입력에 항상 같은 출력(결정론), **표면 텍스트에 영어 오행 키나 "생/극" 같은 용어가 없음** 검증.
- `reading-deep.test.ts`: `generateDeepBite(category, count)`가 2~3개 섹션을 반환하고 각 body가 충분한 길이, 결정론적임을 검증.
- 기존 18개 테스트 회귀 유지.
- 수동 QA: 내 사주 저장 후 홈에 오늘의 한 입 노출 / 날짜 바뀌면 내용 변동 / 결과에서 한 입 더 펼침 / 한 상 티저 노출.

## 10. 이후로 미루는 것 (이번 초안 밖)

- 실제 결제·구독·프리미엄 잠금 동작 (🍱 사주 한 상).
- 한 입 더 / 한 상의 **진짜 깊은 풀이 품질** — Phase 2 AI(Claude)에서 생성, 그때 한 상을 잠금.
- 기본 결과 자체의 깊이 강화 (별도 과제).
- "오늘의 사람/음식/색깔" 외 추가 "오늘의 ~~" 항목 확장.
- 데일리 알림/푸시, 연속 출석 등 리텐션 장치.
